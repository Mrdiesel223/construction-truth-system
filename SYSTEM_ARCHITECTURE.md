# 🏗️ Construction Truth System - Technical Documentation

## 1. Executive Summary
The **Construction Truth System** is a high-integrity field staff management platform built specifically for construction contractors (bridges, buildings, and site work). Unlike traditional tracking apps, this system is designed to provide an immutable record of work—linking location, time, and visual evidence into a single "Truth Timeline."

## 2. Core Architecture

### 2.1 Backend Engine (`apps/backend`)
*   **Stack:** Node.js, Express, Prisma ORM, PostgreSQL.
*   **Authentication:** Role-based (ADMIN, FIELD_STAFF, MANAGER) using JWT and Bcrypt hashing.
*   **Intelligence Layer:**
    *   **Geofence Watchdog:** A geospatial utility using the Haversine formula to detect if workers leave assigned site boundaries during active tasks.
    *   **Automated Alerting:** Real-time generation of `GEOFENCE_BREACH` and `OVERDUE_TASK` alerts.
    *   **Reporting Engine:** Aggregates attendance, task completions, and incidents into daily snapshots and audit-ready CSVs.
*   **Storage:** Cloudinary integration for secure, high-performance hosting of site evidence photos.

### 2.2 Mobile Portal (`apps/mobile`)
*   **Stack:** Flutter (Dart), Provider for state management.
*   **Truth Capture Workflow:**
    *   **Passive Tracking:** Background service with a foreground notification that pings GPS every 5 minutes to the backend.
    *   **Active Proof:** Mandatory camera capture at task completion, automatically tagged with high-accuracy GPS coordinates.
*   **Offline Resilience:**
    *   **Local Vault:** SQLite (sqflite) database that caches all pings and visit logs when internet is unavailable.
    *   **Sync Engine:** `SyncService` that monitors connectivity and automatically flushes cached data once site signal is restored.

### 2.3 Admin Intelligence Panel (`apps/admin`)
*   **Stack:** Next.js (App Router), TypeScript, Tailwind CSS, Lucide Icons.
*   **Modules:**
    *   **Tactical Map:** Live visualization of the entire workforce with real-time connectivity status.
    *   **Project Sites:** Portal to define project centers and geofence radii (e.g., "Main Bridge Site - 500m").
    *   **Evidence Vault:** A high-end verification gallery for contractors to review and officially "Verify Truth" of site photos.
    *   **Intelligence Reports:** Chronological daily summaries and incident logs for site compliance and auditing.

## 3. The "Truth" Lifecycle

1.  **Planning:** Contractor defines a **Site Boundary** and assigns a **Task** (e.g., "Curing Pillar A") to a Worker.
2.  **Execution:** Worker launches the app (Automatic Background Tracking begins). Worker clicks **Start Task**.
3.  **Verification:** Worker captures a **Site Photo** upon completion. The app captures the **GPS Coordinate** of the photo.
4.  **Audit:** The **Watchdog** checks if the photo GPS is within the Site Boundary.
5.  **Finalization:** Contractor reviews the photo in the **Evidence Vault** and marks it as **Verified**.
6.  **Reporting:** The system generates a **Daily Summary** showing that the task was completed on time, in the right place, and with verified proof.
